# Versioning Policy

QNSP APIs use path-based versioning (e.g., `/v1/`).

## Stability guarantees

- **Stable**: No breaking changes within a major version
- **Beta**: May change with notice
- **Alpha**: Experimental, may change without notice

## Version lifecycle

1. New versions are introduced alongside existing versions
2. Deprecated versions receive security fixes only
3. End-of-life versions are removed after the deprecation period
